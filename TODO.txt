Tsk 1)
When Mergician is starting up, the frontend should display a loading box that goes away once loading is complete. This box can display startup progress (which should have a blue color theme) and also have a mode to display an error state (in a redd color theme). The updating of this box can be a simple poll done every 3 seconds.

If any configuration or startup error is found, the error should be displayed in this box. When it's in the `DetectTimezone` method to check Gitlab timezone, the stays in the UI should be "Checking Gitlab..." (no need to say specifically what). If the timezone check fails, this indicates a problem with Gitlab. The timezeone check should keep retrying until it works and the frontend should say "Error contacting Gitlab, please contact administrator." and any detected error displayed in the error area of the box, until Gitlab works again and the frontend should recover.

Task 2)
When Mergician is starting up, before checking the Gitlab timezone do a Gitlab health check API call. If this fails, keep retrying and ensure the frontend shows the "Error contacting Gitlab, please contact administrator." error. If the health check endpoint also returns the server time, then this check can be combined with the timezone check.

Task 3)
The GitlabUserFactory.IsServiceTokenConfigured property should be removed. Instead, checking if the service token is configured should be part of Mergician start up. If it isn't, that is an error state and should be displayed in the loading box "Gitlab Service token is not configured". The application should not continue from this state. This means the downstream code doesn't need to check if the service token is configured.

Task 4)
Similar to checking Gitlab health, also check database access and health on startup. If it fails, display this in the loading error box and keep retrying till it works.

Task 5)
Run the integration tests to ensure they still work after all the changes. There is no need to check for the error states above, we only need to know that the happy path still works.
