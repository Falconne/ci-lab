Task 1)
On the merge group details cards, if the header is the MR title, clicking it should go to the MR (same as the explicit MR link). There should also be a smaller subheading with the branch name as the subtitle (along with the existing copy ison). Clicking the branch name should go to the branch as it currently does with this header.

Task 2)
If the merge attempt with the service account fails, the error needs to be shown on the merge group details page. The merge group (and affected branch) status bubbles should say "Blocked" with the reason of "Merge error".

Task 3)
If the automerge fails due to lack of permissions from the service user, this shold be shown in the error and retries should only be done once every 5 minutes.

For other unexpected failures during automerge, the retry should have an exponential backoff up to 5 minutes.

The shortest retry time after a merge failure, no matter what, should be 20 seconds.
