Task 1)
When Mergician is checking for deleted branches, if a branch is not deleted, use the "Compare" API to check if the branch has any differences to merge into the default branch. Otherwise, consider the branch as if it was deleted and remove it from the database. We don't care about branches without changes. When checking a user's activity for new branches, it is ok to add it to the database without check for differences, as it is likely they just pushed to it. The subsequent check for deleted branches will take care of it if it is not active.
