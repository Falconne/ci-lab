Task 1)
In the Merge Group Details page, add 2 new toggle buttons at the top, one called "Auto Merge" and one called "Auto Rebase". Enabling "Auto Merge" should enable "Auto Rebase" automatically. These two settings should be backed by entries in the database in the Merge Group table.

Add a new persisted background service to the Mergician backend called `AutoMergeService`.  This should be created such that it will always be active and not be killed by asp.net. This service should call the Gitlab API using the Gitlab service user token that the backend is configureed with. It should run a loop every 5 seconds to check all merge groups that have either of these settings enabled. For each such merge group:
1) Check if the auto merge setting is enabled:
    * If so, for any branches in the merge group that have a Merge Request created:
        * If so, and the branch is behind the target merge branch in the MR, use the Gitlab API to rebase the branch.
    * Ignore branches without merge requests for this section.
2) Check if auto merge is enabled
    * If so, check if _all_ the branches in the MR are ready to be merged. This means they all should:
        * Have merge requests created.
        * Have all reauired approvals given, if any are required.
        * If there are any Pipelines, the most recent must be complete and sucessful.
        * If auto rebase is enabled, then al branches must be upto date with their target branch.
        * If possible, check from the Gitlab API if each MR is marked as mergeable.
    * If all branches are ready to be merged, then use the Gitlab API to merge _all_ of them in the merge group. If any are not ready, none should merge.
        * Kick off all the merges in parallel so as to have the best chance of them all merging together.
        * If any fail to merge because they became unmergable in between the last check ands the merge attmept, that is ok, we will have a chance to try again in future loops of this background service. However log a warning in the backend and also add a warning to be shown in this merge group's details that some branches got merged early.
